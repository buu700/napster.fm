En cuanto a la música, yo soy esencialmente sólo agarrando contenido en YouTube basado en metadatos (títulos, artistas, etc.) de la base de datos de Discogs.

¿Así que todo el servicio es dependiente en menores ineficiencias en el sistema de detección de piratería de YouTube?

Shhh... Probablemente no hacen el daño, pero no, no necesariamente; no ser demasiado selectivos sobre qué versión de una canción que buscas.

Cool. Así que ¿por qué Peer.fm vagamente parecen Windows 8?

Windows 8 presentó algunas ideas interesantes y un UX casi pasable, pero en su mayor parte simplemente creo el nuevo estilo de Metro / tendencia plana parece fresco.

También, me gustó la idea de replanteo Napster – originalmente un Windows 95 escritorio interfaz – como una aplicación Web moderna 8-ish para Windows.

Como se mencionó en la página de inicio, todo se desarrolla al aire libre en GitHub; cualquier mano extra sería brillante. Correo electrónico me o algo antes de enviar cualquier solicitud de tirón, pero por favor no dude en entrar peticiones ni informes de fallos en el sistema de temas.

¿Si decido debo contribuir, qué detalles técnicos sobre Peer.fm debo tener en cuenta?

1. Para gestionar todos nuestros datos, estoy utilizando el almacén de datos de "servidor" en tiempo real fantástico Bases avanzadas (YC S11). En consecuencia, aparte de un conjunto de reglas de validación de bases avanzadas, esto es pura ingeniería del lado del cliente; hackers de grado A JS solamente.

2. Que el SDK de bases avanzadas y la API de YouTube, las bibliotecas principales y estoy usando los marcos son: Cierre de Google, AngularJS, Azúcar y Manos a la obra (son todo brillantes).

3. Si estás acostumbrado a recorriendo el árbol DOM con jQuery, mi ClosureQuery basadas en el cierre puede parecer un poco familiar.

¿Lo que específicamente se podría utilizar ayuda?

Para prefacio, estoy abierto a cualquier contribución propuesta al proyecto. Dicho esto:

1. Automatización de pruebas: Peer.fm actualmente tiene cero cobertura de prueba y no tengo ningún plan para escribir cualquier yo mismo en un futuro próximo; Esto podría ser un proyecto para alguien para manejar.

2. Ilustraciones: actualmente estoy prestado el logo de Akonadi para el favicon de Peer.fm (ninguna significación especial; KDE es simplemente impresionante).

3. Documentación: no hay ninguno en este momento.
