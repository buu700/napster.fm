Lalu bagaimana cara kerjanya?

Sejauh musik streaming, saya 'm pada dasarnya hanya meraih konten yang ada di YouTube berdasarkan metadata (judul, artis, dll) dari Discogs database.

Jadi seluruh layanan bergantung pada kecil inefisiensi dalam sistem deteksi pembajakan YouTube?

Shhh... Mereka mungkin tidak terluka, tapi tidak, belum tentu; hanya jangan terlalu selektif tentang versi lagu Anda setelah.

Dingin. Jadi mengapa Apakah Peer.fm samar-samar menyerupai Windows 8?

Windows 8 memperkenalkan ide-ide menarik dan UX hampir lumayan, tetapi untuk sebagian besar aku hanya berpikir gaya Metro baru / datar tren terlihat keren.

Juga, saya menyukai gagasan reimagining Napster-awalnya Windows 95 desktop antarmuka-sebagai Windows 8-ish modern aplikasi Web.

Berbicara tentang desktop, dimana ada aplikasi saya asli dan mobile? Kapan saya dapat menjalankan ini di pemanggang roti saya?

Menantikan; Saya punya beberapa ide tentang hal itu. Jika Anda benar-benar peduli, hanya menembak saya email dan saya akan memberitahu Anda segera setelah saya memiliki update.

Siapa kamu?

Saya alias online adalah buu700, tapi aku paling sering pergi oleh Ryan Lester IRL. Saya seorang mahasiswa di Carnegie Mellon, saat ini melepas beberapa tahun bekerja di SpaceX dan melakukan hal-hal lain. Di waktu luang saya, saya berkontribusi untuk w3sec.

Saya ingin membuat permintaan kecil Anda / spam Anda / meminta Anda keluar pada tanggal / memberi Anda uang membawa Anda ke lapangan / mengirimkan komentar. Bagaimana saya dapat menghubungi Anda?

Yakin, tidak masalah; hanya Gulir ke bawah untuk informasi kontak saya.

Saya tidak ada lebih baik untuk dilakukan dengan hidupku; Bagaimana saya dapat berkontribusi untuk Peer.fm?

Seperti yang disebutkan pada home page, segala sesuatu yang dikembangkan di tempat terbuka di GitHub; tangan tambahan apapun akan brilian. Email saya atau sesuatu sebelum mengirim permintaan tarik apapun, tapi jangan ragu untuk log semua laporan bug atau permintaan fitur di masalah sistem.

Jika saya harus memutuskan untuk berkontribusi, apa teknis spesifik mengenai Peer.fm saya harus menyadari?

1. Untuk mengelola semua data kami, saya menggunakan fantastis real-time data "serverless" toko Firebase (YC S11). Sebagai akibatnya, selain dari seperangkat aturan validasi Firebase, ini adalah murni teknik sisi klien; Grade-A JS hacker hanya.

2. Selain dari Firebase SDK dan YouTube API, Perpustakaan besar dan kerangka kerja saya menggunakan adalah: Google penutupan, AngularJS, Gula, dan Bootstrap (mereka semua brilian).

3. Jika Anda digunakan untuk melintasi DOM dengan jQuery, saya berbasis penutupan ClosureQuery mungkin tampak agak akrab.

Apa khusus Anda bisa menggunakan bantuan?

Pengantar, saya terbuka untuk setiap kontribusi yang diusulkan untuk proyek. Yang sedang berkata:

1. Otomatis pengujian: Peer.fm saat ini memiliki cakupan uji nol dan aku tidak berencana untuk menulis setiap diri dalam waktu dekat; ini bisa menjadi sebuah proyek untuk orang lain untuk menangani.

2. Artwork: Aku sedang meminjam logo Akonadi untuk Peer.fm favicon (tidak ada makna khusus; KDE di hanya awesome).

3. Pengguna dokumentasi: tidak ada pada saat ini.
