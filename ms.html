Maka bagaimana kerja-kerja ini?

Setakat streaming muzik, saya saya pada dasarnya hanya merebut kandungan sedia ada di YouTube yang berdasarkan metadata (tajuk, artis, dll) dari pangkalan data Discogs.

Jadi Perkhidmatan keseluruhan adalah bergantung kepada ketidakcekapan kecil dalam YouTube di cetak rompak-pengesanan sistem?

Shhh... Mereka mungkin tidak mencederakan, tetapi tiada, tidak semestinya; hanya Jangan bersikap terlalu memilih mengenai versi mana lagu anda selepas.

Sejuk. Jadi mengapa tidak Peer.fm kelihatan menyerupai Windows 8?

Windows 8 memperkenalkan beberapa idea-idea yang menarik dan yang UX hampir agak baik, tetapi bagi sebahagian besar aku hanya berfikir gaya Metro baru / trend rata kelihatan sejuk.

Selain itu, saya suka idea reimagining Napster – asalnya sebuah Windows 95 desktop antaramuka – sebagai Windows 8-ish moden Aplikasi Web yang.

Di mana bercakap desktop, Adakah aplikasi asli dan telefon bimbit saya? Bilakah boleh saya jalankan ini pada saya pembakar roti?

Harap sabar menunggu; Saya mempunyai beberapa idea-idea tentang itu. Jika anda benar-benar peduli, hanya menembak saya e-mel dan saya akan memberitahu anda sebaik sahaja saya mempunyai maklumat terkini.

awak ni siapa?

Saya alias talian adalah buu700, tetapi saya sering pergi oleh Ryan Lester IRL. Saya seorang pelajar di Carnegie Mellon, sedang pesat beberapa tahun untuk bekerja di SpaceX dan melakukan hal-hal lain. Dalam masa saya percuma, saya menyumbang kepada w3sec.

Saya ingin membuat pertanyaan yang kecil anda / spam anda meminta anda keluar pada tarikh / memberikan anda wang / membawa anda ke Mahkamah / menghantar komen. Bagaimana saya boleh menghubungi anda?

Pasti, tiada masalah; Tatal ke bawah hanya untuk maklumat hubungi saya.

Perlu apa-apa lebih baik lakukan dengan hidup saya; Bagaimana saya boleh menyumbang kepada Peer.fm?

Seperti yang dinyatakan di halaman rumah, segala-galanya dibangunkan di buka di GitHub; mana-mana tangan tambahan akan cemerlang. E-mel saya atau sesuatu sebelum menghantar sebarang permintaan tarik, tetapi sila tidak teragak-agak untuk masuk mana-mana laporan pepijat atau permintaan ciri isu-isu sistem ini.

Jika saya perlu memutuskan untuk menyumbang, khusus apa teknikal mengenai Peer.fm saya harus sedar?

1. Untuk menguruskan semua data kami, saya menggunakan stor data masa nyata "serverless" yang hebat yang Firebase (YC S11). Akibatnya, selain satu set kaedah-kaedah pengesahan Firebase, ini adalah tulen-klien Kejuruteraan; Gred-A penggodam JS sahaja.

2. Selain daripada Firebase SDK dan YouTube API, Perpustakaan Utama dan rangka kerja yang saya menggunakan adalah: Penutupan Google, AngularJS, Gula, dan Bootstrap (mereka semua cemerlang).

3. Jika anda sedang digunakan untuk terabas DOM dengan jQuery, ClosureQuery berasaskan penutupan saya mungkin kelihatan agak biasa.

Apa khusus anda boleh menggunakan bantuan?

Untuk preface, saya terbuka kepada apa-apa sumbangan yang dicadangkan untuk projek tersebut. Bahawa yang berkata:

1. Automatik ujian: Peer.fm kini mempunyai liputan ujian sifar dan saya bercadang untuk menulis apa-apa yang sendiri dalam masa terdekat; ini boleh menjadi satu projek bagi orang lain untuk mengendalikan.

2. Karya seni: saya buat masa ini saya meminjam logo Akonadi untuk favicon Peer.fm (tiada kepentingan khas; KDE yang hanya menggerunkan).

3. Pengguna dokumentasi: Tiada buat masa ini.
